Column Ordering

Click the left/right toggle to reorder the columns in the rendered table. This does not change the query order as determined in the Fields|Rearrange section of the Views UI.

Combining Columns

To have the contents of one column combined with another (showing only the header title of the first), position the combinee to the right of the source column and click "Merge left." You may then supply separator text (e.g. a dash, a space, etc.). Note that html is allowed in this field. To have the second field appear under the first, enter <br>.

Select "Inline title" to have the field title (label) displayed in the table.

Alignment

This will change the alignment of both the column header and the data in the table.

Width

You may specify a width for each column. This width will be accommodated if possible by the browser.

  • Width may be specified in: px, em, rem, %, ch, ex.
  • Widths will be maintained for different screen sizes if possible.
  • If necessary, the width will be reduced below that specified in order to display all columns if possible.

Click-sorting

On field types that allow sorting, you may enable click sorting, along with the sort order that will be applied on the first click.

Check 'Default sort' to force the table to be displayed with a given column initially activated —or: select 'No default'.

Hide if empty

Select this if you want the table to render without this column in the case where the views query produces no data for this field.

Borders

Border thickness may be specified in: px, em, rem, %, ch, ex

Add views row classes

Add the default row classes like views-row-1 to the output.

  • Deselect this to quickly reduce the amount of markup the view provides. Your theme may rely on these classes for styling, so be sure to check the rendering of your tables to see if this setting is needed.

Row class

The class to provide on each row.
